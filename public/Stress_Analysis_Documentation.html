1. Project Overview

Human Stress Analysis Between Normal and ADHD Patient Using Neural Network Based on EEG Signals is a university research project investigating how stress and cognitive response manifest differently in the brain activity of normal subjects versus ADHD patients. The study uses controlled emotional stimuli to elicit EEG responses, which are then processed through a signal processing and machine learning pipeline to build a discriminative patient profile.

The core methodology applies the Russell Circumplex Model of Affect — mapping brain signals into a 2D Valence-Arousal emotional space — to study whether ADHD patients exhibit measurably distinct patterns from normal subjects across emotional states including Calm, Happy, Sad, Fear, and cognitive tasks (Eyes Open, Eyes Closed, Arithmetic).

Key Capabilities

  • Multi-Stimulus EEG Analysis — 7 controlled stimulus conditions applied to both patient groups
  • Full-Spectrum Band Decomposition — Elliptic filters extract Delta, Theta, Alpha, Beta, and Gamma bands from all 19 EEG channels
  • MFCC Feature Extraction — Mel-frequency cepstral coefficients applied to each frequency band, borrowed from speech processing for robust spectral feature capture
  • Valence-Arousal Neural Network — Two-output MLP maps EEG features to emotional state coordinates on the circumplex model
  • Patient Profiling — Per-band neural network models trained to classify emotional state from brain activity patterns
  • MATLAB Signal Processing Toolkit — End-to-end pipeline using MATLAB's Signal Processing and Neural Network toolboxes

2. System Architecture & Flowchart

The system follows a linear signal processing pipeline: raw EEG recordings are loaded per-stimulus per-group, decomposed into frequency bands, features are extracted from each band, assembled into per-band feature matrices with emotional labels, then fed to a two-output MLP that learns to map brain activity to Valence-Arousal coordinates.

5. Processing Pipeline (Detailed)

Step 1 — Data Loading & Preprocessing

FinalCoding_UPDATED.m lines 1–29Data

All 14 CSV files (7 stimuli × 2 groups) are loaded using MATLAB's readtable(). Note: the script runs with the working directory set to the group-specific data folder (either Data/ADHD/ or Data/Normal/), so file paths are relative. The first 4 metadata rows are removed and the time column (column 1) is dropped, leaving a pure 19-column signal matrix per stimulus.

%% Load all 7 stimulus CSVs
Calm       = readtable("Calm.csv");
Sad        = readtable("Sad.csv");
Happy      = readtable("Happy.csv");
Fear       = readtable('Fear.csv');
EO         = readtable('EO.csv');   % Eyes Open
EC         = readtable('EC.csv');   % Eyes Closed
Arithmetic = readtable('Arithmetic.csv');

%% Remove 4-row metadata header
Calm(1:4,:) = [];  % (and same for all 7)

%% Drop timestamp column, keep 19 EEG channels (cols 2–20)
Calm = Calm(:, 2:20);  % (and same for all 7)

Each resulting table has ~13,099 rows × 19 columns. Individual electrode columns are then extracted as numeric arrays via table2array() using named variables (e.g., CFP1 = Calm / FP1 channel, SFP1 = Sad / FP1, etc.).

Step 2 — Frequency Band Splitting

splitband.m · Called from FinalCoding_UPDATED.m lines ~188–343Signal

splitband() is called on every individual channel array across all 7 stimuli (19 channels × 7 conditions = 133 calls per group). It uses MATLAB's ellipord and ellip functions to design minimum-order elliptic filters, then filter() to apply them. The filter cutoffs reference John (1980) EEG band definitions.

function [Delta, Theta, Alpha, Beta, Gamma, DtoG] = splitband(sig)

% Delta: 0–4 Hz (lowpass)
[N, Wp] = ellipord(4/125, 5/125, 0.1, 60);
[B, A]  = ellip(N, 0.1, 60, Wp);
Delta   = filter(B, A, sig);

% Theta: 4–8 Hz (bandpass)
[N, Wp] = ellipord([4 8]/125, [2 10]/125, 0.1, 60);
[B, A]  = ellip(N, 0.1, 60, Wp);
Theta   = filter(B, A, sig);

% Alpha: 8–13 Hz, Beta: 13–30 Hz, Gamma: 30–50 Hz  (same pattern)

% Broadband DtoG: 0–50 Hz (lowpass)
[N, Wp] = ellipord(50/125, 51/125, 0.1, 60);
DtoG    = filter(B, A, sig);
Filter Design Note
The normalized cutoff frequencies are expressed as fractions of the Nyquist frequency (125 Hz for a 250 Hz signal). Elliptic filters are chosen because they achieve the sharpest roll-off for a given filter order, critical for cleanly separating narrow EEG bands (e.g., Theta at 4–8 Hz and Alpha at 8–13 Hz share a boundary). Passband ripple = 0.1 dB, stopband attenuation = 60 dB.

Step 3 — MFCC Feature Extraction

melcepst.m · Called from FinalCoding_UPDATED.mSignal

Mel-frequency cepstral coefficients (MFCCs) — a feature technique originally developed for speech recognition — are applied to each frequency-band sub-signal. This captures the spectral envelope of the band signal in a compact, perceptually-weighted feature vector. The key parameters used throughout are fixed:

% For every band of every channel of every stimulus:
CextFP1G1 = melcepst(Cs1ec_bandsG1, 125, 'M', 12, 14, 1024, 93);
%                    signal          fs    type  nc  p    n     inc
%  fs  = 125 Hz (sampling rate passed to melcepst)
%  'M' = magnitude spectrum
%  nc  = 12 cepstral coefficients to return
%  p   = 14 mel filterbank channels
%  n   = 1024-point FFT window
%  inc = 93-sample frame increment (hop size)

Each call returns a matrix of shape [num_frames × 12]. With ~13,099 samples, a 1024-point window, and 93-sample hop, this produces approximately 140 frames per channel-band-stimulus combination, yielding a 140×12 MFCC matrix per call.

Why MFCC for EEG?
MFCC was borrowed from speech processing because EEG band signals share important properties with audio: they are non-stationary time series whose local spectral shape is more informative than raw amplitude. MFCCs on filtered EEG bands capture how the spectral content of each brainwave band evolves over time, providing a compact feature set that generalizes better than raw sample values.

Step 4 — Feature Matrix Assembly

FinalCoding_UPDATED.m lines ~1700–1815ML Prep

After MFCC extraction, all channel-level feature matrices for a given band and stimulus are vertically concatenated into a single per-stimulus, per-band matrix. These are then stacked across stimuli into the final band-level training matrices (AATHETA, AALPHA, ABETA, ADELTA) that feed the neural network. Rows represent feature frames; columns 1–12 are MFCC features.

%% Concatenate all 19 channels for Arithmetic / Theta band
AATHETA = [AextFP1T1; AextFP2T2; AextFP3T3; AextFP4T4; AextFP5T5;
           AextFP6T6; AextFP7T7; AextFP8T8; AextFP9T9; AextFP10T10;
           AextFP11T11; AextFP12T12; AextFP13T13; AextFP14T14; AextFP15T15;
           AextFP16T16; AextFP17T17; AextFP18T18; AextFP19T19];

%% Transpose applied twice (net effect: returns to original orientation)
AATHETA = AATHETA';
AATHETA = AATHETA';
Double Transpose Note
The script applies AATHETA = AATHETA' twice in succession (a comment notes "am too lazy to put ' everywhere"). This is a no-op — transposing twice returns to the original shape. The matrix retains its [frames × 12] orientation before label columns are appended.

Step 5 — Valence-Arousal Label Encoding

FinalCoding_UPDATED.m lines ~1816–1918ML Prep

Two additional columns (13 and 14) are appended to each band matrix representing the expected Valence (positive/negative affect) and Arousal (high/low energy) scores for that stimulus. This follows the Russell Circumplex Model, placing each emotion in a 2D space:

% Happy: high valence (+1), high arousal (+1)
HATHETA(:,13) = 1;   HATHETA(:,14) = 1;

% Calm: high valence (+1), low arousal (–1)
CATHETA(:,13) = 1;   CATHETA(:,14) = -1;

% Sad: low valence (–1), low arousal (–1)
SATHETA(:,13) = -1;  SATHETA(:,14) = -1;

% Fear: low valence (–1), high arousal (+1)
FATHETA(:,13) = -1;  FATHETA(:,14) = 1;

% Eyes Open / Closed / Arithmetic: neutral (0, 0)
EATHETA(:,13) = 0;   EATHETA(:,14) = 0;

The four emotional stimuli are then stacked into the final training matrix. Baseline stimuli (EO, EC, Arithmetic) are excluded from the combined training set — they serve as baseline/reference conditions only:

EmotionATHETA = [HATHETA; CATHETA; FATHETA; SATHETA];

Step 6 — Data Randomization

FinalCoding_UPDATED.m lines ~1920–1936 · random.mML Prep

Before training, the combined emotion matrix rows are shuffled using randperm() to prevent the neural network from learning ordering artifacts (all Happy rows first, then Calm, etc.).

[rowT, colT] = size(EmotionATHETA);
RT = randperm(rowT);
randomSigT = EmotionATHETA(RT, :);   % shuffled full matrix

% random.m provides the same as a reusable function:
function [randomSig] = random(sig1)
    [row] = size(sig1);
    R = randperm(row);
    randomSig = sig1(R,:);

Step 7 — MLP Neural Network Training

MLPVA.m · Called from FinalCoding_UPDATED.mNeural Network

The MLPVA function trains a Multi-Layer Perceptron using MATLAB's Neural Network Toolbox (newff API). The randomized feature matrix is split: columns 1–12 are the input features, columns 13–14 are the Valence-Arousal targets. The same split structure is used for both the training set (randomSigT) and the test set (AATHETA).

function [y, tgtestS, perAcc, out, ts, perf, time, netVA] = MLPVA(randomSigT, AATHETA)

trainS   = randomSigT(:, 1:12);     % input features (12 MFCC per frame)
tgtrainS = randomSigT(:, 13:14);    % target: [Valence, Arousal]
testS    = AATHETA(:, 1:12);        % test input
tgtestS  = AATHETA(:, 13:14);       % test targets

net = newff(minmax(trainS'), [10, 10, 2], ...
            {'tansig', 'tansig', 'purelin'}, 'traingdx');
net.trainParam.lr     = 0.001;      % learning rate
net.trainParam.epochs = 10000;      % max iterations
net.trainParam.show   = 100;        % display interval
net.trainParam.goal   = 0.01;       % SSE convergence target

[net, tr] = train(net, trainS', tgtrainS');  % train
y = sim(netVA, testS');                       % test inference

Step 8 — Evaluation & Visualization

MLPVA.m lines 30–39 · plotVA2.mOutput

Accuracy is computed as the percentage of test frames where the absolute difference between the predicted and target output is below a threshold of 1.2 (reflecting a correct classification on the ±1 scale):

out = abs(tgtestS' - y);    % absolute error per output neuron
accuracy = 0;
for j = 1:length(out)
    if (out(j) < 1.2)
        accuracy = accuracy + 1;
    end
end
perAcc = accuracy / length(out) * 100;  % percentage accuracy

The training loss curve is plotted with semilogy(tr.epoch, tr.perf) (MSE vs. epoch on a log scale). The Valence-Arousal output is visualized via plotVA2, which applies LOESS smoothing and overlays the two outputs in red (Valence) and blue (Arousal):

function [y1, y2] = plotVA2(y)
    y1 = smooth(y(1,:), 0.1, 'rloess');   % Valence output (red)
    y2 = smooth(y(2,:), 0.1, 'rloess');   % Arousal output (blue)
    plot(y1, 'Color', 'red', 'LineWidth', 2);
    hold;
    plot(y2, 'Color', 'blue', 'LineWidth', 2);

7. Valence-Arousal Emotion Model

The project uses the Russell Circumplex Model of Affect (1980) to represent emotional states in a continuous 2D space rather than discrete categories. This is particularly relevant for brain-computer interface research as it allows the neural network to output a position in emotional space rather than a class label, enabling gradient-based analysis of how ADHD affects emotional processing.

😊 Happy

Valence: +1    Arousal: +1

High positive affect, high energy. Top-right quadrant. Associated with elevated Beta and Gamma activity.

😌 Calm

Valence: +1    Arousal: −1

Positive affect, low energy. Top-left quadrant. Associated with elevated Alpha band.

😨 Fear

Valence: −1    Arousal: +1

Negative affect, high energy. Bottom-right quadrant. Elevated Beta frontal activity.

😢 Sad

Valence: −1    Arousal: −1

Negative affect, low energy. Bottom-left quadrant. Elevated Theta/Delta, reduced Alpha.

The three baseline conditions (Eyes Open, Eyes Closed, Arithmetic) are labelled (0, 0) — the neutral center of the circumplex. They are included in individual band matrices but excluded from the combined EmotionA* training matrix, so the neural network is trained only on the four emotionally-labelled states.

Toolbox Compatibility
The newff function is part of the legacy Neural Network Toolbox API. In newer versions of MATLAB (R2019b+), it was superseded by feedforwardnet. The script uses the older API; if running on a newer MATLAB, ensure the Neural Network Toolbox (now Deep Learning Toolbox) is installed and that the legacy function layer is available. The core functionality remains the same.
Running the Pipeline
Set the MATLAB working directory to either Data/ADHD/ or Data/Normal/ before running FinalCoding_UPDATED.m, as all CSV paths are relative. Ensure all .m files in Code/ are on the MATLAB path. The script runs end-to-end without user input — training outputs (MSE curve, V-A plot, accuracy) appear automatically on completion.
